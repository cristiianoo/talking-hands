# 🤝 Como Contribuir

Obrigado pelo interesse em ajudar a "Talking Hands"!

## Regras Básicas
1. Abre um **Issue** antes de começares a programar uma funcionalidade grande.
2. Cria um fork do projeto e faz um branch com o nome da funcionalidade: `feature/nova-funcionalidade` ou `fix/correcao-bug`.
3. Certifica-te que o código compila e não tem erros de Lint.

## Padrão de Commits
Utilizamos o padrão Conventional Commits:
- `feat:` Nova funcionalidade
- `fix:` Resolução de bug
- `docs:` Alterações na documentação
- `style:` Formatação, ponto e vírgula, etc.
- `refactor:` Refatorização de código
